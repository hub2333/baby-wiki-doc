# AGENTS.md

## 语言

- 默认用中文交流和写项目说明。

## 环境

- 当前办公电脑受 Windows 透明加密软件影响，读写本仓库时优先使用 WSL Ubuntu：`wsl -d Ubuntu-22.04 -u root`。
- 这个限制只针对有透明加密影响的机器。普通 Windows 电脑可以直接用 Windows 工具或 Obsidian 操作本仓库。
- 无论使用 Windows 还是 WSL，都应保持 `.gitattributes` 定义的 LF 行尾策略。

## 项目目标

- 用 Karpathy 的 LLM Wiki 思路，把育儿知识整理成 Obsidian 可浏览、可追溯来源、可持续扩展的双链知识库。
- 当前主体是 0-3 岁育儿知识，正在扩展 3-12 岁儿童成长、健康和家庭教育主题。

## 目录约定

- `index.md`：总入口。
- `00-入口与导航/`：首页、规划和导航。
- `10-年龄阶段/`：年龄阶段页。
- `20-主题知识/`：主题页、高风险边界页。
- `90-来源索引/`：来源总表和来源规划。
- `95-来源笔记/`：具体来源摘要。
- `99-项目管理/`：建设日志和维护记录。

## 页面约定

- 每个 Markdown 页面都应包含 frontmatter，至少包括 `title`、`status`、`updated`。
- 主题页优先包含：核心结论、分阶段建议或判断路径、高优先级提醒、主要依据、参考来源、延伸阅读。
- 来源笔记优先包含：机构、来源类型、原始链接、适用年龄、支撑页面、来源摘要。
- 使用 Obsidian 双链连接年龄页、主题页、来源索引和来源笔记。

## 内容安全

- 医疗、急救、用药和发育筛查内容只做信息整理，不替代医生诊疗。
- 高风险页面应保留明确的红旗信号和就医/急救提示。
- 不要把单一来源的建议写成适用于所有孩子的绝对规则。

## 变更习惯

- 新增或大幅调整内容时，同步更新相关入口页、来源页和建设日志。
- 修改大量文件前先确认是否只是行尾或格式噪音。
- 不要为了整理工作区而回滚用户未明确要求回滚的内容。
