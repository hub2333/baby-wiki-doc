# baby-wiki-doc

这是一个用 Karpathy 的 LLM Wiki 思路和 Obsidian 双链组织的育儿知识库。当前主体是 0-3 岁婴幼儿照护，并已开始扩展 3-12 岁儿童成长、健康和家庭教育主题。

## 当前结构

- `index.md`：总入口。
- `00-入口与导航/`：年龄段首页、阶段规划和全局导航。
- `10-年龄阶段/`：0-3 岁年龄阶段页。
- `20-主题知识/`：主题页和高风险边界页。
- `90-来源索引/`：权威来源总表和扩展来源规划。
- `95-来源笔记/`：来源摘要、适用年龄和支撑页面。
- `99-项目管理/`：建设日志、模板、后续任务和项目维护记录。

## 写作原则

- 先用权威公开来源建立骨架，再把信息整理成中文可浏览笔记。
- 主题页面保持“核心结论、分阶段建议、高优先级提醒、主要依据、参考来源、延伸阅读”的结构。
- 来源笔记承接“原始来源 -> 摘要 -> 支撑页面”的中间层，尽量写清适用年龄、来源机构和原始链接。
- 新增主题页优先使用 [[99-项目管理/主题页模板]]；新增来源笔记优先使用 [[99-项目管理/来源笔记模板]]。
- 医疗、急救和用药内容只做信息整理，不替代医生诊疗；遇到红旗信号应建议联系医生、急诊或当地急救服务。

## 本地使用

用 Obsidian 打开仓库根目录即可浏览双链。为了减少跨电脑换行符噪音，本仓库通过 `.gitattributes` 统一 Markdown 和配置文件为 LF 行尾。

如果某台 Windows 电脑有透明加密或同步软件导致直接读写异常，可以在 WSL 中操作项目；普通 Windows 环境不需要强制使用 WSL。

## 下一步重点

1. 固定跨 Windows/WSL 的行尾策略，避免无意义 diff。
2. 继续把 3-12 岁内容从规划推进为可浏览的首版主题页。
3. 对高风险医疗/急救页面做质量审计，减少过强表述。
4. 提升来源笔记质量，补充限制、冲突和更新时间。
